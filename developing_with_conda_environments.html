Developing code within conda environments

Posted on Wed 15 April 2020 in misc

Here we'll discuss using conda environments, and installing a package in developer mode so you can edit it and immediately have the changes propagated. A good discussion is here.

Start a new conda environment

Create a new environment:

conda create --name myenv
conda activate /path/to/myenv

Download cw_search_manager code from github

See here for a discussion of using on your computers and ssh keys and such.

git clone https://git.ligo.org/unimelb/cw_search_manager.git
cd cw_search_manager
pip install -e .

Note that the pip install -e . runs python setup.py --install and the -e flag means it is in "developer" mode. Instead of copying the code to your site-packages it points your python path to this directory so that any changes you make are automatically propagated to your conda environment. This can be dangerous in general, but is quite useful when developing code.
